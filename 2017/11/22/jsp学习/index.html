javabean中scope的不同值的作用范围

  1. application 整个服务器
  2. session 整个回话
  3. request 可以使用服务器内重定向进行获得值
  4. page 仅此页面

注意细节

  1. request提交时的中文乱码问题解决方法。 <% request.setCharacterEncoding = "utf-8"%>但是在tomcat8.0版本之后，中文乱码问题已经自动解决。
  2. 利用response对象时默认输出到网页第一行。利用out.flush()清楚缓冲区内容来将out对象要输出到内容输出到屏幕上。
  3. jsp:useBean 中scope的不同值所代表的范围.(都可用<jsp:getProperty>获得属性值)
    • application:整个服务器(可以用application.getParmater()获得值)
    • session: 整个用户的回话(可以用sessionn.getParmater()获得值)
    • request: 此次的请求范围内,(要分清请求重定向和服务器内部转发)即服务器的内部跳转到的页面是有效的(request.getDispatcher.forward(request,response)).也可以使用<jsp:forward>.不可以用超链接<a href="xxx.jsp"></a>.因为超链接和response.sendRedirect()都是请求重定向.
    • page: 只有这个页面本身,在此页面上可以用pageContext.getParameter()获得属性值.
